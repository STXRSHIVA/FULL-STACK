Front-end Development
The client-facing part of web development that involves creating and designing the user interface (UI) and user experience (UX) using HTML, CSS, and JavaScript.
HTML (HyperText Markup Language)
The standard markup language used for creating the structure of web pages and their content.
CSS (Cascading Style Sheets)
Stylesheets used to control the layout and appearance of HTML elements on a webpage.
JavaScript
A versatile programming language used to create interactive and dynamic elements on the webpage, enabling a better user experience.
Back-end Development
The server-side development involving the logic, databases, and server operations necessary to power the application and manage data.
Server
The central system that hosts and manages the application, processes requests, and returns appropriate responses to the client.
Database
A structured collection of data that stores information, which can be retrieved, updated, or deleted as needed by the application.
Backend Frameworks (e.g., Node.js, Django, Ruby on Rails)
Pre-built frameworks that streamline server-side development by providing tools and libraries to expedite the process of creating server logic and APIs.
API (Application Programming Interface)
A set of rules and protocols that allows different software applications to communicate and interact with each other, enabling data exchange and functionality access.
Version Control (e.g., Git)
A system for tracking and managing changes in the source code during development, facilitating collaboration and enabling the ability to revert to previous versions.
Deployment
The process of making a web application available and accessible for users after development, often involving configuring servers, setting up databases, and optimizing performance.
DevOps (Development and Operations)
The practice of combining software development and IT operations to enhance the efficiency, collaboration, and automation in the development and deployment process.
